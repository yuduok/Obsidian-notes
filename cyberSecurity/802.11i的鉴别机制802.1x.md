
801.11i的安全性基于以下特点：
- 基于用户身份认证
- 和AP建立安全关联，并在建立安全关联时分配临时密钥TK
- 802.1x完成双向的、基于用户的身份认证

#### 802.1X 认证框架

基本组件：
- 客户端（请求者）
- 认证服务器（通常是 RADIUS）
- 接入点（认证器）

认证流程：
```plaintext
客户端        接入点        认证服务器
   |             |               |
1. 连接请求 ---->|               |
   |             |               |
2. <---- EAP请求 |               |
   |             |               |
3. EAP响应 ----->|               |
   |             |--- EAP转发 -->|
   |             |               |
4. <-- 认证结果 --|               |
   |             |               |
5. 生成会话密钥 <-----------------
```

#### EAP 认证方法

常见 EAP 类型：
- PEAP（受保护的 EAP）
- EAP-TLS（传输层安全）
- EAP-TTLS（隧道传输层安全）
- EAP-SIM（SIM 卡认证）
